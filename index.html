Vigil Orchestrator

Serverless event-driven backbone — routes, deduplicates, and optimizes every alert
CORE RESPONSIBILITIES
  • Ingests Slack alert events via API Gateway + SQS FIFO (strict ordering, no duplicates)
  • Fingerprints alerts and checks DynamoDB for cached RCAs (71% hit rate) or known issues
  • Routes to AI Agent only when fresh analysis is needed — saving cost and time
  • Relays Agent responses back to the correct Slack thread
  • Tracks alert frequency and triggers escalation when thresholds are breached
TECH STACK
AWS Lambda SQS FIFO DynamoDB API Gateway SSM Parameter Store SAM (IaC)

Vigil AI Agent

Agentic AI that investigates alerts like a senior engineer would
CORE RESPONSIBILITIES
  • Receives alert context from the Orchestrator and plans an investigation strategy
  • Queries Coralogix logs via MCP to find error patterns around the alert timeframe
  • Scans relevant GitHub code via MCP to trace the root cause in source
  • Generates a structured RCA with evidence, confidence levels, and remediation steps
  • Uses a skills-based modular architecture — new capabilities are added as skills
TECH STACK
ECS Fargate Claude Sonnet 4.6 AWS Bedrock Coralogix MCP GitHub MCP Arize (Evals)

Key Features
What Vigil delivers to on-call teams

Automated RCA

Reads Coralogix logs + GitHub code around the alert timeframe. Posts structured RCA with evidence and confidence levels in the Slack thread (~2 min).

Smart Caching

71% cache hit rate on recurring alerts = instant response. DynamoDB-backed fingerprinting with 15-day staleness. Click 'Analyse' anytime for a fresh look.

Known Issue Tagging

Engineers mark alerts as known issues with Jira links and custom comments. Future occurrences skip analysis entirely, reducing noise.

Frequency Escalation

Auto @mentions channel managers when an alert fires repeatedly beyond configured thresholds. Configuration-driven, per-channel control.
